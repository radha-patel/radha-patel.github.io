Who Will Be The Next Popstar?

6.08 Interconnected Embedded Systems Final Project

We created an interactive, Who Will be the Next Popstar? game in which users play a series of minigames to test their dancing, singing, and rhythmical prowess and receive scores that represent their musical ability. The minigames include

  • a Just Dance-esque game in which different steps are depicted on an LCD screen in sequence that the user has to then copy in time;
  • a karaoke game where the objective is for the user to sing as closely to the original recording as possible;
  • and finally, a rhythm game in which users will receive a unique sequence of buttons/actions to hit in time with music and will be scored based on the timing of the moves.
The scoreboard for all players is visible on a self-refreshing website along with playing instructions. The purpose of this project is to help individuals overcome their inhibitions and anxiety towards musical performance as well as relieve stress from the associated endorphin rush. We describe the specifics below.

My part of the project was the Just Dance mini-game, in which a song 1-2 minutes in length is played through the speaker and different dance moves are depicted one after another on their LCD screen, which the user then has to perform. The Esp32 is considered the “remote” (held in the right hand) and an accelerometer is used to detect if the user is performing the correct motion and if so, count the number of repetitions. A score is awarded for each individual dance move based on the number of correct repetitions of a move that the user performs and how this number compares to the expected count. The individual scores are totalled as the song progresses for each user and the final amount is the overall score that is displayed and sent to the server via an HTTP POST request. The database and public scoreboard then updates accordingly. I found that the most consistent approach to detecting different dance moves was to use a distinctive features approach in which I broke down each motion into two-dimensions and then found several (sequential) characteristic features in the acceleration along the two relevant axes. I established thresholds based on the features and constructed a state machine that determines which part of the motion is being completed and increments a counter once an entire motion has been completed (i.e. all the thresholds have been passed).
